# 1-Week Launch Checklist

## Day 1: Conversion Path

- [ ] README top section explains the problem in 10 seconds.
- [ ] README includes install, quickstart, before/after, generated files, score output, and safety model.
- [ ] Social card renders correctly from `docs/marketing/assets/social-card.svg`.
- [ ] Demo script is ready.

## Day 2: Publish Readiness

- [ ] Package names verified on npm.
- [ ] npm access to `@agentnav` scope confirmed.
- [ ] `license`, `repository`, `bugs`, `homepage`, `keywords`, `files`, and `publishConfig.access` present.
- [ ] `pnpm build`, `pnpm test`, `pnpm lint`, and `pnpm test:e2e` pass.
- [ ] `npm pack --dry-run` checked for every package.
- [ ] Publish `0.1.0` packages publicly.

## Day 3: Demo And Proof

- [ ] Record 90-second demo using `docs/marketing/demo-script.md`.
- [ ] Capture proof snippet: booking action requires confirmation.
- [ ] Capture proof snippet: `/llms.txt` generated from config.
- [ ] Capture proof snippet: CLI score returns grade.

## Day 4: Pre-Launch Outreach

- [ ] Send 30 targeted feedback messages using `docs/marketing/outreach.md`.
- [ ] Post teaser copy from `docs/marketing/social-posts.md`.
- [ ] Track feedback objections in GitHub Discussions or a launch notes doc.

## Day 5: Soft Launch

- [ ] Post to GitHub/npm, LinkedIn, X/Bluesky, and relevant React/Next communities.
- [ ] Ask for technical critique, not generic promotion.
- [ ] Patch README based on repeated confusion.

## Day 6: Main Launch

- [ ] Submit Hacker News post from `docs/marketing/social-posts.md`.
- [ ] Submit Product Hunt only if the demo asset and maker comment are ready.
- [ ] Reply to technical comments for 12 hours.

## Day 7: Follow-Up

- [ ] Publish blog post draft from `docs/blog/how-to-make-a-nextjs-site-readable-by-ai-agents.md`.
- [ ] Create issues using labels from `.github/labels.yml`.
- [ ] Open roadmap discussion using `.github/DISCUSSION_TEMPLATE/roadmap.md`.
- [ ] Ask early users for example sites to showcase.

## Success Metrics

- [ ] 300+ GitHub stars.
- [ ] 1,000+ npm downloads across packages.
- [ ] 20+ GitHub issues/discussions from real developers.
- [ ] 5+ external examples, mentions, or demos.
- [ ] 3+ contributors or serious PRs.
